# 5052 - 백준 

## 🔗 문제 링크
- https://www.acmicpc.net/problem/5052

## 💡 접근 방법 및 핵심 로직

**[문제 조건]**
- t : tc 수 (1 <= t <= 50)
- n : 전화번호 수 (1 <= n <= 10,000)
- l : 전화번호 길이 (1 <= l <= 10)

**[풀이]**

- 각 전화번호들 서로가 다른 전화번호들의 접두어가 아니어야 함.
- 서로 비교하는 로직일 경우 $n^2$ 의 시간 복잡도가 발생
- tc 수와 모든 전화번호 길이도 고려 할 경우 $tc * n^2 * l$ 즉 최악의 경우 $50 * 100,000,000 * 10$ 번의 연산 발생 => 시간초과 

- 대신 Trie 알고리즘으로 Tree 만들면 
- insert : O(m), search : O(m)
- 길이순 역정렬 후 모든 단어를 순회하면서 해당 단어가 다른 단어의 접두사가 되는지 판단 후 접두사가 된다면 종료, 모두 순회를 해도 접두사가 되지않는다면 pass 
- $n * log(n) * l + n * l * 2$ 
- $10,000 * 14 * 10 + 10,000 * 10 * 2 = 1,400,000 + 200,000 = 1,600,000$
- $1,600,000 * 50 = 80,000,000$


## ⏱️ 복잡도
- **시간 복잡도:** $n * log(n) + n * (log(l) * 2)$

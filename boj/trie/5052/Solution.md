# 5052 - 백준 

## 🔗 문제 링크
- https://www.acmicpc.net/problem/5052

## 💡 접근 방법 및 핵심 로직

**[문제 조건]**
- t : tc 수 (1 <= t <= 50)
- n : 전화번호 수 (1 <= n <= 10,000)
- l : 전화번호 길이 (1 <= l <= 10)

**[풀이]**

- 각 전화번호들 서로가 다른 전화번호들의 접두어가 아니어야 함.
- 서로 비교하는 로직일 경우 $n^2$ 의 시간 복잡도가 발생
- tc 수와 모든 전화번호 길이도 고려 할 경우 $tc * n^2 * l$ 즉 최악의 경우 $50 * 100,000,000 * 10$ 번의 연산 발생 => 시간초과 

- Trie 자료구조 도입
- insert : O(m) (m : 문자열 길이)
- search : O(m) (m : 문자열 길이)
- tc 당 들어오는 모든 문자열을 받고 insert 만 수행
- insert 하다가 이미 존재하는 접두사 단어를 만났을 경우 return false
- insert 로직 돌다가 자신보다 더 긴단어가 있을 경우 return false
- 그외 return true
- 50 * 10,000 * 10 = 5,000,000 번 연산

## ⏱️ 복잡도
- **시간 복잡도:** $O(tc * n * l)$
